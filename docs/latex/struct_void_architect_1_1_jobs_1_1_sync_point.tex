\doxysubsection{Detailed Description}
Thread-\/safe synchronization point for job dependency management. 

\doxylink{struct_void_architect_1_1_jobs_1_1_sync_point}{Sync\+Point} provides a robust mechanism for coordinating job execution dependencies in a lock-\/free manner. It supports atomic counter-\/based synchronization with status propagation and continuation management.\hypertarget{struct_void_architect_1_1_jobs_1_1_sync_point_autotoc_md0}{}\doxysubsection{\texorpdfstring{Key features\+:}{Key features\+:}}\label{struct_void_architect_1_1_jobs_1_1_sync_point_autotoc_md0}

\begin{DoxyItemize}
\item Atomic dependency counting with status aggregation
\item Lock-\/free hot path for counter operations
\item Hybrid continuation storage (inline + overflow)
\item Automatic failure/cancellation cascade propagation
\item Debug-\/friendly naming and timing information
\end{DoxyItemize}\hypertarget{struct_void_architect_1_1_jobs_1_1_sync_point_autotoc_md1}{}\doxysubsection{\texorpdfstring{Architecture\+:}{Architecture\+:}}\label{struct_void_architect_1_1_jobs_1_1_sync_point_autotoc_md1}

\begin{DoxyItemize}
\item Hot data (counter, status) uses atomic operations for lock-\/free access
\item Continuations use hybrid storage \+: 6 inline slots + overflow vector
\item Status can only degrade (Success -\/\texorpdfstring{$>$}{>} Failed/\+Cancelled) for safety
\item Thread-\/safe continuation management with minimal locking
\end{DoxyItemize}\hypertarget{struct_void_architect_1_1_jobs_1_1_sync_point_autotoc_md2}{}\doxysubsection{\texorpdfstring{Usage example\+:}{Usage example\+:}}\label{struct_void_architect_1_1_jobs_1_1_sync_point_autotoc_md2}

\item Failed/\+Cancelled\+: Status degrades atomically (first failure wins)
